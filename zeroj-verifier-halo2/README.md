# zeroj-verifier-halo2

Halo2 proof verification via Rust FFM bindings.

This module verifies Halo2 proofs using the zcash/halo2 Rust library, accessed through Java's Foreign Function & Memory (FFM) API. Halo2's key advantage is **no trusted setup** — it uses transparent IPA (Inner Product Argument) commitment parameters.

## Features

- **No trusted setup** required (transparent IPA parameters)
- Supports Pallas curve (`CurveId.PALLAS`)
- Verifies proofs generated by the zcash/halo2 proving stack
- Regenerates VK from params for verification (zcash halo2 v0.3 compatible)
- Native Rust library loaded via Java FFM

## Key Types

| Type | Description |
|------|-------------|
| `Halo2Verifier` | FFM-based verifier — parses base64 proof/params JSON, delegates to Rust |
| `Halo2Library` | FFM foreign function bindings to Rust |
| `Halo2NativeLoader` | Classpath/system library loading |

## Prerequisites

The module requires the native Rust library to be compiled:

```bash
cd zeroj-verifier-halo2/halo2-rust
cargo build --release
```

The compiled library (`libzeroj_halo2.dylib`/`.so`) must be available on the library path or classpath.

## Usage

```java
var verifier = new Halo2Verifier();
var registry = VerifierRegistry.empty();
registry.register(verifier);

var orchestrator = new VerifierOrchestrator(registry, vkRegistry);
VerificationResult result = orchestrator.verify(halo2Envelope, material);
```
